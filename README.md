# 🥦 Food Supply App

A React-based web application that allows users to browse nearby food suppliers, search and filter products by category, manage a shopping cart, and get instant help via a built-in chatbot assistant — all from a clean, responsive interface.

🌐 **Live at:** [http://www.cloud-itgen.de](http://www.cloud-itgen.de)

---

## Features

- **Product Browsing** — View a grid of food products with images, supplier names, and prices
- **Category Filtering** — Filter products by category: Vegetables, Fruits, Dairy, Grains, Meat, Seafood
- **Live Search** — Search products by name or supplier in real time
- **Cart Management** — Add items to cart with a live item count displayed in a fixed bottom bar
- **Chatbot Assistant** — A floating chat widget with auto-reply support for product, price, supplier, and delivery queries

## Getting Started (Local Development)

### 1. Clone the repository

```bash
git clone <your-repo-url>
cd food-supply-app
```

### 2. Install dependencies

```bash
npm install
```

> If you encounter a Vite version conflict, pin the correct versions:
> ```bash
> npm pkg set devDependencies.vite="4.4.5" devDependencies.@vitejs/plugin-react="4.0.3"
> rm -rf node_modules package-lock.json
> npm install
> ```

### 3. Start the development server

```bash
npm run dev
```

The app will be available at **http://localhost:5173** by default.

## Adding New Products

Products are defined in `src/data/products.js`. To add a new product, append an entry to the array:

```js
{
  id: 9,
  name: "Cherry Tomatoes",
  supplier: "Sunrise Farms",
  price: "$4.00 / kg",
  category: "Vegetables",
  img: "https://your-image-url.com/photo.jpg"
}
```

Valid category values: `Vegetables`, `Fruits`, `Dairy`, `Grains`, `Meat`, `Seafood`

## Chatbot Auto-Reply

The chatbot widget is accessible via the 💬 button fixed at the bottom-right of the screen. It responds automatically to common user queries.

| Topic | Example Query |
|-------|---------------|
| Greeting | `hi`, `hello` |
| Categories | `vegetables`, `fruits`, `dairy`, `meat`, `seafood`, `grains` |
| Pricing | `price`, `cost`, `cheap` |
| Suppliers | `supplier` |
| Cart & Orders | `cart`, `order` |
| Delivery | `delivery`, `shipping` |
| Farewell | `bye`, `thanks` |

To add new auto-reply rules, edit the `autoReply` function in `src/components/ChatBot.jsx`:

```js
if (text.includes("organic"))
  return "🌱 All our vegetables are organically sourced from certified farms!";
```

## Running with Docker (Local)

Make sure Docker is installed — [Download](https://docs.docker.com/get-docker/)

### Build the image

```bash
docker build -t food-supply-app .
```

### Run the container

```bash
docker run -d -p 80:80 food-supply-app
```

Open **http://localhost** in your browser.

### Useful Docker commands

```bash
docker ps                        # list running containers
docker stop <container-id>       # stop a container
docker rm <container-id>         # remove a container
docker rmi food-supply-app       # remove the image
```

> ⚠️ If port 80 is occupied by Apache2 or nginx, stop them first:
> ```bash
> sudo systemctl stop apache2
> sudo systemctl disable apache2
> ```

## Deployment — AWS EC2 + nginx

The app is deployed on an AWS EC2 instance running Ubuntu, served via nginx.

### 1. SSH into your EC2 instance

```bash
ssh -i your-key.pem ubuntu@your-ec2-public-ip
```

### 2. Install Node.js 18 on the server

```bash
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt install -y nodejs
```

Verify:

```bash
node --version
npm --version
```

### 3. Install nginx

```bash
sudo apt install -y nginx
```

### 4. Clone the repository

```bash
git clone <your-repo-url>
cd <your-project-directory>
```

### 5. Install dependencies and build

```bash
npm install
npm run build
```

This generates the `dist/` folder which nginx will serve.

### 6. Configure nginx

```bash
sudo nano /etc/nginx/sites-available/default
```

Replace the contents with:

```nginx
server {
    listen 80;
    server_name yourdomain.com www.yourdomain.com;

    root /home/ubuntu/<your-project-directory>/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

> The `try_files` directive is critical for React single-page apps — without it, page refreshes will return a 404.

### 7. Test and restart nginx

```bash
sudo nginx -t
sudo systemctl restart nginx
```

### 8. Open port 80 in EC2 Security Group

In the AWS Console go to:
**EC2 → Security Groups → Inbound Rules** and ensure the following ports are open:

| Port | Protocol | Purpose |
|------|----------|---------|
| 22 | TCP | SSH access |
| 80 | TCP | HTTP web traffic |
| 443 | TCP | HTTPS web traffic (if using SSL) |

### 9. (Recommended) Add HTTPS with Certbot

```bash
sudo apt install certbot python3-certbot-nginx -y
sudo certbot --nginx -d yourdomain.com -d www.yourdomain.com
```

Verify auto-renewal:

```bash
sudo certbot renew --dry-run
```

## Deployment Options Comparison

| Option | Best For | Pros | Cons |
|--------|----------|------|------|
| **EC2 + Docker** (current) | Containerised deployments | Portable, consistent environment | Manual server management |
| **EC2 + nginx** | Full control | Flexible, customisable | Manual server management |
| **AWS S3 + CloudFront** | Static sites | Cheap, global CDN, no server | Frontend only |
| **AWS Amplify** | React apps | Auto-deploy from GitHub, HTTPS built-in | Less control |
